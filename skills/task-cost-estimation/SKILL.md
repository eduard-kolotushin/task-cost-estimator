# Оценка трудозатрат (TaskTracker wiki)

## Когда опираться на навык

Запрос на **оценку задачи**, **декомпозицию с оценками**, **таблицу оценок** в wiki или сценарий «загрузить задачу по коду → оценить → создать wiki-страницу с таблицей».

## Инструменты

1. **get_task_definition(code)** — загрузить **задачу (юнит)** `GET /rest/api/unit/v2/{code}`; использовать `_formatted_task_context` и поля ответа (не wiki-плагин).
2. **get_wiki_page(code)** — wiki-плагин; для **проверки** созданной страницы с оценками (код из ответа create), не для исходной задачи.
3. **get_wiki_hierarchy(spaces?)** — дерево wiki в пространстве(ах); в API всегда `root=null` (полный обзор). Для навигации по структуре страниц.
4. **create_wiki_page_estimation** — создать **новую** страницу с таблицей; передать `summary`, `space`, массив `rows`.
5. **update_wiki_page** — только если нужно поправить тело существующей страницы (редко).
6. **link_wiki_parent_child(parent, child)** — иерархия wiki: дочерняя страница под родителем. Если в запросе есть **[Система]** с кодом родителя (`single-run --parent-page`), после успешного create вызови с `child` = `code` из ответа API создания.

## Таблица: 5 колонок

| Колонка | Правило |
|--------|---------|
| **Команда** | Один из **блоков** эталонной таблицы (как в `example_wiki_page_est_table.txt`): **Аналитика/Проектирование**, **Разработка**, **Тестирование**, **Документирование**. |
| **Компонент** | **Всегда** `VIEW` в каждой строке данных. |
| **Этап** | **Только** строка из **справочника этапов** ниже — не выдумывать свои названия (не подставлять вместо этапа абстрактные «RND», «Аналитика» и т.п.). |
| **Оценка** | Только **чел.-дни**. **Сумма** оценок подзадач в колонке **Декомпозиция** в этой строке. |
| **Декомпозиция** | Поле `dekompozitsiya` в API: **несколько строк текста** — каждая строка превращается в **пункт нумерованного списка** в wiki. В строках — подзадачи с оценкой в **чел.-дни** и **обоснование** набора. |

**Как попадает в wiki:** модель **генерирует** многострочный `dekompozitsiya`; при вызове **create_wiki_page_estimation** тело страницы собирается в JSON редактора (**TipTap/ProseMirror**, `type: doc`, таблица в `wiki_page_body`): заголовок, секции по **Команда**, строки данных, колонка «Декомпозиция» — нумерованный список в ячейке, в конце — **Итого**. Порядок объектов в `rows` может быть любым — строки **группируются по Команда** перед отрисовкой.

**Справочник этапов** (колонка **Этап**; имена этапов совпадают с колонкой «Этап» в `example_wiki_page_est_table.txt`; файл показывает старый трёхколоночный макет, в инструменте — пять колонок):

- **Аналитика/Проектирование:** Проектирование UI; Тех. проектирование; Прототип/RND.
- **Разработка:** Разработка ПМИ/Наполнение тестовой модели; Разработка (UI); Разработка (Back); Разработка (DevOps); Отладка и DEV-тестирование; Внутрикомандное демо и исправления по замечаниям.
- **Тестирование:** Ручное тестирование (ST, IFT); Разработка и прогон автотестов (ST, IFT).
- **Документирование:** Документирование.

**Выбор строк:** по смыслу задачи реши, **какие из этих этапов** реально нужны; для каждого выбранного этапа — одна строка таблицы; в **`dekompozitsiya`** дай список подзадач (по одной на строку). Этапы без работы можно не включать или дать **0** чел.-дней, если требуется полная сетка как в примере.

Сумма подзадач (чел.-дни) в колонке **Декомпозиция** должна совпадать с **Оценка**.

## Порядок действий

1. Вызвать **get_task_definition** по коду задачи, которую нужно оценить.
2. Сопоставить работу со **справочником этапов**; выбрать подмножество этапов и для каждого заполнить строку (поля **Команда** = блок, **Этап** = точное имя из справочника).
3. Вызвать **create_wiki_page_estimation** с новым заголовком `summary` и **`rows` — JSON-массив объектов** (поля `komanda`, `komponent`, `etap`, `otsenka`, `dekompozitsiya`), не строку с Python-синтаксисом.
4. При необходимости связи с родителем — **link_wiki_parent_child** (см. инструмент п. 6).
5. Опционально: **get_wiki_page** по коду созданной страницы — валидация таблицы в wiki.
6. Сообщить пользователю код созданной страницы из ответа API.

## Не делать

- Не ставить в **Компонент** ничего кроме `VIEW`.
- Не подставлять в **Этап** / **Команда** произвольные ярлыки вместо имён из справочника.
- Не использовать часы вместо чел.-дней.
- Не заполнять колонку **Декомпозиция** (поле `dekompozitsiya`) без явной связи подзадач и обоснования набора.
- Не передавать `rows` одной строкой с одинарными кавычками — только массив объектов в вызове инструмента.
